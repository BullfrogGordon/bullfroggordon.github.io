StoryRound- Collaborative Multiplayer Game

This is a digital version of a game I used to play with students. Creative writing is fun! It's more fun to collaborate and write a story with friends. This game lets you write with a group of friends no matter where you all are!

How it works:

A .NET API written in C# stores a unique game PIN, the names of the game players, and the stories they are writing. Each player starts a story and then the story is sent to the server and given to another player to continue. This process is repeated until all of the rounds are over and the completed stories are presented to all of the players.

QuickRoster- The Ad-Hoc Roster Creation and Selection App

Another teaching inspired app, this simple interface allows the creation of multiple rosters of users. Once a roster is created, you can head over to Choosers to randomly select a user from the list. You can also generate an entire list in random order, or randomly select users one-at-a-time until everyone has had a turn!

How it works:

QuickRoster was designed and written to be entirely client-side. It stores all user-created rosters in browser memory as a single object. The choosers are written in simple Javascript, making the app itself lightweight and ready to be packaged as an installed application.

Check out QuickRoster here: QuickRoster

Chess Opening Trainer

Want to impress your friends with your vast knowledge of chess openings and their strange and esoteric names? Try this naming quiz!

How it works:

A Flask API written in python grabs a database entry from a MongoDB instance. The entry includes the name of the opening along with the FEN, a chess position identifer. The name of the opening gets sent to the front-end. The user positions the board and submits. The FEN and name are sent to the API. The server compares the POST to the corresponding database entry and responds with whether or not they match. The board and pieces are from the open source chessboardjs written by Chris Oakman.
